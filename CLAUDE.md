## Project Overview

This is the **C-slop documentation website** repository. C-slop is a token-minimal programming language designed for web applications, optimizing for machine efficiency by reducing token count while maintaining expressiveness. The language compiles to JavaScript/Node.js and uses symbols (`@`, `$`, `#`, `>`) instead of keywords to eliminate boilerplate.

The documentation site is built with **Docusaurus v3.9.2** (a React-based static site generator) and uses TypeScript.

## Commands

### Development
```bash
cd docs
npm start          # Start dev server at http://localhost:3000 with live reload
npm run serve      # Serve production build locally
npm run typecheck  # Run TypeScript type checking
```

### Build & Deploy
```bash
cd docs
npm run build      # Generate static site into build/ directory
npm run clear      # Clear Docusaurus cache and generated files
npm run deploy     # Deploy to GitHub Pages (requires GIT_USER env var or USE_SSH=true)
```

### Docusaurus-Specific
```bash
cd docs
npm run swizzle    # Customize Docusaurus components
npm run write-translations     # Extract i18n translation strings

```

### Key Configuration Files

**docusaurus.config.ts** - Main configuration:
- Site metadata: title "C-slop", tagline "Write web apps in 80% fewer tokens"
- URL: https://c-slop.dev
- Dark mode by default with system preference support
- Blog is disabled (blog: false)
- Documentation editable at: https://github.com/c-slop/c-slop/tree/main/docs/
- Prism syntax highlighting with bash and json support
- Navigation includes Docs, Examples, Playground

**sidebars.ts** - Uses autogenerated sidebar from docs/ folder structure

### Content Organization

Documentation content lives in `docs/docs/` and is automatically organized by the file structure. The sidebar is autogenerated. To add new documentation:

1. Create `.md` or `.mdx` files in `docs/docs/`
2. Use frontmatter for metadata (title, sidebar position, etc.)
3. Sidebar updates automatically based on file structure

### React Components

The homepage (`src/pages/index.tsx`) is a custom React component. Other custom components go in `src/components/`.

To modify the homepage features section, edit `src/components/HomepageFeatures/`.

### Styling

- Global CSS: `src/css/custom.css`
- Component-scoped: `*.module.css` files (CSS modules)
- Dark theme: Dracula (Prism), configurable in docusaurus.config.ts

## Language Specification Reference

The `C-SLOP.md` file in the repository root contains the complete language specification for C-slop. Key concepts:

- **Symbols over keywords**: `@` (database), `$` (request/input), `#` (response/output), `>` (pipe), `?` (query), `!` (action)
- **Token efficiency**: 75-82% token reduction vs JavaScript for web operations
- **Built-in web primitives**: Routes, database, HTML rendering as first-class features
- **Pipeline-first**: Data flows through `>` operators
- **Implicit inference**: Types, returns, async all inferred

This specification should guide documentation content when writing about the language syntax and features.
